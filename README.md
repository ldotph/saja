# САЖА

Сайт с афишами локальных концертов, заявками музыкантов, панелью
администратора и экспериментальным разделом релизов недели.

## Запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`.

## Где редактируются афиши

Новые афиши редактируются через панель администратора:

```text
/saja-vault-74q9m
```

Логин и пароль задаются в `.env` на сервере:

```text
ADMIN_LOGIN=sazhafm
ADMIN_PASSWORD=ratusha
ADMIN_SESSION_SECRET=replace-with-long-random-string
SUBMISSION_RETENTION_DAYS=40
```

Файл [data/events.ts](/Users/mg/Desktop/saja/data/events.ts) пока содержит стартовые афиши, которые были на сайте до появления панели.

Изображения теперь разложены по папкам:

- логотипы: `/Users/mg/Desktop/saja/assets/logos`
- афиши: `/Users/mg/Desktop/saja/assets/posters`

Для каждой афиши в этом массиве можно менять:

- `title`
- `date`
- `dateLabel`
- `city`
- `venue`
- `mapUrl`
- `poster`
- `actions`
- `priorityClass`

## Как меняется приоритет

Поле `priorityClass` принимает значения `1`, `2` или `3`.

Сортировка работает так:

1. Сначала по `date` по возрастанию.
2. Внутри одинаковой даты по `priorityClass`: `1`, потом `2`, потом `3`.

## Форма и заявки

Форма валидирует поля и отправляет `multipart/form-data` на `/api/inquiry`.
Заявки сохраняются в локальное хранилище сайта и видны в разделе `Заявки`
панели администратора.

Поля заявки:

- город
- дата
- название события
- клуб
- ссылка на билеты
- ссылка на встречу VK, если есть
- Telegram ID заявителя
- файл афиши

Заявки хранятся 40 дней, затем автоматически очищаются при следующем обращении
к хранилищу.

## Релизы недели

Публичный раздел релизов находится на URL:

```text
/releases
```

На главной странице ссылки на него нет. Релизы создаются и редактируются в
панели администратора. Пользователи могут ставить оценку от 1 до 5, рейтинг
считается автоматически. Для лидербордов используется взвешенная формула,
которая учитывает и среднюю оценку, и количество голосов. Повторные голоса
ограничиваются cookie и серверным хэшем пользователя. Перед отправкой оценки
пользователь вводит контрольное слово `САЖА`; проверка выполняется на сервере и
не требует внешней капчи.

## Что нужно подставить вручную

- `ADMIN_SESSION_SECRET` в `.env` на сервере.
- `RELEASE_VOTE_SECRET` для защиты хэшей голосования.
- Финальный публичный домен сайта.
