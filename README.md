# SAJA Booking

Одностраничный сайт концертного агентства на `Next.js + TypeScript`.

## Запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`.

## Где редактируются афиши

Новые афиши редактируются через панель администратора:

```text
/ratusha-desk-7x4p
```

Логин и пароль задаются в `.env` на сервере:

```text
ADMIN_LOGIN=sazhafm
ADMIN_PASSWORD=ratusha
ADMIN_SESSION_SECRET=replace-with-long-random-string
```

Файл [data/events.ts](/Users/mg/Desktop/saja/data/events.ts) пока содержит стартовые афиши, которые были на сайте до появления панели.

Изображения теперь разложены по папкам:

- логотипы: `/Users/mg/Desktop/saja/assets/logos`
- афиши: `/Users/mg/Desktop/saja/assets/posters`

Для каждой афиши в этом массиве можно менять:

- `title`
- `date`
- `dateLabel`
- `city`
- `venue`
- `mapUrl`
- `poster`
- `actions`
- `priorityClass`

## Как меняется приоритет

Поле `priorityClass` принимает значения `1`, `2` или `3`.

Сортировка работает так:

1. Сначала по `date` по возрастанию.
2. Внутри одинаковой даты по `priorityClass`: `1`, потом `2`, потом `3`.

## Форма и заявки

Форма валидирует поля и отправляет `multipart/form-data` на `/api/inquiry`.
Заявки сохраняются в локальное хранилище сайта и видны в разделе `Заявки`
панели администратора.

Поля заявки:

- город
- дата
- название события
- клуб
- ссылка на билеты
- ссылка на встречу VK, если есть
- Telegram ID заявителя
- файл афиши

Заявки хранятся 60 дней, затем автоматически очищаются при следующем обращении
к хранилищу.

## Что нужно подставить вручную

- `ADMIN_SESSION_SECRET` в `.env` на сервере.
- Финальный публичный домен сайта.
